# 创始人团队数字执行 Showcase

这份文档用于展示 AgentSpace 与纯“人机协作通讯平台”的差异。

目标不是证明多个 Agent 能在一个频道里聊天，而是证明 AgentSpace 能把一支小团队的碎片原料，转成可上岗的数字员工，并让它们持续推进跨天工作。

## 一句话脚本

> 给系统一批零散业务原料，招募合适的数字员工，让它们在频道、任务、文档和交接流里完成一次真实的跨天执行闭环。

## 适合演示的对象

- 1-10 人创始团队
- 交付 / 代运营 / 小型工作室
- 需要同时处理客户沟通、内容产出、执行推进和风险兜底的小团队

## 演示主题

推荐主题：`48 小时活动落地战`

你需要准备 4 类原料：

1. 活动 brief
2. 场地或客户约束
3. 预算说明
4. 历史复盘、聊天摘要或会议纪要

## 标准演示路径

### Step 1：导入业务原料

把上面的文件、摘要或手工输入导入工作区。

演示重点：

- 原料不是直接塞进一条长对话
- 系统能记录来源、状态和后续引用关系

### Step 2：生成候选员工

根据原料生成至少 3 个候选员工，例如：

- 总控协调员
- 内容策划
- 执行运营
- 风险审核

演示重点：

- 员工有来源、有职责、有适配场景
- 候选员工和已上岗员工是两种状态

### Step 3：招募上岗并加入频道

把需要的员工招募到实际协作体系中，让它们进入不同频道或默认挂到总控室。

演示重点：

- 这是“组织补位”，不是单纯把一个 bot 拉进群
- 上岗后，成员状态会同步到协作侧

### Step 4：创建跨天任务

在频道中发起真实任务，例如：

> 48 小时内完成活动方案、物料清单、风险清单和对外宣传初稿。先拆任务，再分配负责人。

演示重点：

- 任务绑定频道与执行者
- 频道内能同时看到消息、任务和当前成员

### Step 5：制造阻塞与升级

在任务执行中途插入变化，例如：

- 预算砍掉 30%
- 场地限制新增
- 客户要求增加传播动作
- 某个角色缺少上下文或权限

演示重点：

- 任务不是继续在聊天里漂浮
- 系统能看到阻塞、升级、交接与下一手负责人

### Step 6：沉淀正式输出

要求系统最终产出：

- 活动执行方案
- 任务分工表
- 风险清单
- 宣传文案初稿
- 待确认事项列表

演示重点：

- 输出不只是一串聊天记录
- 结果能够沉淀成文档、附件、任务状态和后续交接基线

## 成功标准

一次成功的 showcase 至少应该让观众看到：

- 原料如何转成数字员工
- 员工如何进入真实工作流
- 任务如何跨步骤推进而不是停留在单轮问答
- 中间阻塞如何触发升级或交接
- 最终结果如何沉淀为长期资产

## 不要这样演示

以下演示方式会让 AgentSpace 看起来像普通的多 Agent 协作工具：

- 让多个 Agent 在频道里自由聊天
- 主要展示角色设定和聊天风格
- 只展示即时回复，不展示任务状态变化
- 只展示群聊记录，不展示文档、附件和交接沉淀

## 推荐口播

可以直接使用下面这段：

> 这不是一个让多个 Agent 一起聊天的演示，而是一个让数字员工进入创始人团队真实工作流的演示。我们先导入业务原料，再生成候选员工，把需要的角色招募上岗，然后让它们在频道、任务和文档流中持续推进跨天工作。中途如果出现预算变化、信息缺失或角色能力边界，系统会通过升级、交接和正式沉淀继续向前推进，而不是把一切留在聊天窗口里。
